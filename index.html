    Audio proxy configuration

    Twitch's CDN blocks direct audio fetching from web browsers (CORS). A small local proxy unlocks audio features. Run it in PowerShell while using the app.

    Setup instructions:
    1. Install Node.js (LTS) if you don't have it
    2. Download proxy.js (provided separately)
    3. Save it anywhere (e.g., Documents\stream-analyser)
    4. Open PowerShell in that folder and run node proxy.js
    5. Leave the PowerShell window open, paste the proxy URL below

    Upload a local recording

    If the live audio probe fails or you want to analyze your OBS recording directly, upload it here. More accurate since it's your original audio before Twitch re-encodes.

    Browser size limit: Chrome can't decode files larger than ~2 GB in a single pass — a full-stream OBS recording will fail. If your recording is big, extract just the audio track first (fast, lossless, produces a file ~10% the size):
    ffmpeg -i "your-recording.mp4" -vn -c:a copy audio.m4a
    Don't have ffmpeg? Install with winget install Gyan.FFmpeg in PowerShell, or download from gyan.dev/ffmpeg/builds.
